# Phase 2 测试指南

## 概述

Phase 2 已完成配置管理系统的增强和日志系统的完善。本文档描述如何测试这些功能。

## 测试前准备

### 1. 构建项目

打开命令提示符（CMD）或 PowerShell，进入项目目录：

```bash
cd D:\java\ClaudeCode\eyesharp
```

执行构建：

```bash
dotnet restore
dotnet build --configuration Debug
```

预期输出：
```
还原成功...
生成成功...
    0 个警告
    0 个错误
```

### 2. 清理测试环境

删除现有配置文件以模拟首次启动：

```bash
del config.json
del config.backup.json
del config.corrupted.*.json
```

## 测试场景

### 场景 1：首次启动 - 密码设置

**目的**：测试首次启动时的密码设置流程

**步骤**：
1. 确保 `config.json` 不存在
2. 运行程序：`dotnet run`
3. 应弹出"设置密码"对话框

**验证点**：
- [ ] 对话框标题："设置密码 - EyeSharp"
- [ ] 显示说明文字："强制休息模式需要密码保护..."
- [ ] 密码和确认密码输入框正常
- [ ] 输入少于4字符时，"确定"按钮禁用
- [ ] 两次密码不一致时，"确定"按钮禁用
- [ ] 显示错误提示："密码长度必须在4-20字符之间"
- [ ] 显示错误提示："两次输入的密码不一致"
- [ ] 密码设置成功后进入主窗口
- [ ] `config.json` 文件被创建

**配置文件示例**：
```json
{
  "RestIntervalMinutes": 30,
  "RestDurationSeconds": 20,
  "IsForcedMode": true,
  "RestWindowMode": "black",
  "CustomImagePath": "",
  "PasswordHash": "10000:BASE64SALT:BASE64HASH",
  "AutoStart": false,
  "LogLevel": "INFO"
}
```

### 场景 2：配置损坏恢复

**目的**：测试配置文件损坏时的自动恢复机制

**步骤**：
1. 先运行程序一次，生成有效的 `config.json`
2. 破坏配置文件：`echo "invalid json" > config.json`
3. 再次运行程序：`dotnet run`

**验证点**：
- [ ] 弹出"配置恢复"对话框
- [ ] 对话框显示警告图标（⚠）
- [ ] 显示默认配置说明（休息间隔30分钟、休息时长20秒、强制模式启用）
- [ ] 点击"确定"后程序继续运行
- [ ] 坏掉的配置被备份：`config.corrupted.YYYYMMDD_HHMMSS.json`
- [ ] 使用默认配置继续运行
- [ ] 主窗口正常显示

**验证备份文件**：
```bash
dir config.corrupted.*.json
```

### 场景 3：日志功能

**目的**：测试日志记录、滚动和清理功能

**步骤**：
1. 运行程序
2. 检查日志目录

**验证点**：
- [ ] `logs/` 目录被创建
- [ ] 当天日志文件：`eyesharp-YYYY-MM-DD.log`
- [ ] 日志内容包含正确的时间戳（北京时间）
- [ ] 日志包含不同的级别：INFO、WARN、ERROR

**检查日志内容**：
```bash
type logs\eyesharp-*.log
```

**预期日志格式**：
```
2025-02-13 14:30:00.1234 INFO eyesharp.App 应用程序启动
2025-02-13 14:30:00.2345 INFO eyesharp.Services.ConfigService 配置加载成功
2025-02-13 14:30:05.3456 INFO eyesharp.App 首次启动密码设置成功
```

**验证点**：
- [ ] `config.backup.json` 被创建

### 场景 5：日志级别切换

**目的**：测试日志级别的动态切换功能

**步骤**：
1. 编辑 `config.json`，修改 `"LogLevel": "DEBUG"`

3. 检查日志输出

**验证点**：
- [ ] 日志级别被正确设置为 DEBUG
- [ ] 日志中包含 DEBUG 级别的日志条目

## 常见问题排查

### 问题 1：程序无法启动

**症状**：运行 `dotnet run` 后立即退出

**排查步骤**：
1. 检查单实例检测：确保没有其他实例在运行
2. 查看日志文件：`logs\eyesharp-*.log`
3. 检查 .NET 版本：`dotnet --version` 应显示 8.x.x

### 问题 2：配置文件未创建

**症状**：密码设置后没有 `config.json`

**排查步骤**：
1. 检查目录权限：确保程序有写入权限
2. 查看日志：查找配置保存错误
3. 手动创建测试：检查是否为权限问题

### 问题 3：日志文件未创建

**症状**：`logs/` 目录不存在或为空

**排查步骤**：
1. 检查 NLog.config 是否正确
2. 检查日志级别设置
3. 确认程序有创建目录权限
