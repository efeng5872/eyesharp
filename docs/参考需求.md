## 1. 项目概述

### 1.1 项目名称

eyesharp - Windows 10/11 护眼提醒程序

### 1.2 项目目标

开发一款运行于 Windows 10/11 平台的护眼软件，通过定时提醒用户休息、强制休息等功能，帮助用户养成良好的用眼习惯，预防视疲劳。

### 1.3 目标用户

*   长时间使用电脑的办公人员
*   程序开发者
*   游戏玩家
*   所有需要长时间注视屏幕的用户

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 定时提醒

*   **功能描述**: 用户可设置休息间隔时间，到达时间后触发提醒
*   **具体要求**:
    *   支持设置分钟数（范围：1-120分钟）
    *   默认值：30分钟
    *   倒计时显示在主窗口
    *   距离下次休息的实时倒计时
    *   休息间隔时间变化后，距离下次休息的实时倒计时会自动刷新
    *   程序启动，自动开始倒计时
    *   休息窗口结束，继续自动倒计时
    *   **修改休息间隔时间且保存后，倒计时将重新从新设置的间隔时间开始**

#### 2.1.4 休息时长

*   **功能描述**: 设置强制休息的持续时间
*   **具体要求**:
    *   可设置范围：5秒 - 300秒
    *   默认值：20秒
    *   休息窗口显示实时倒计时（分秒格式，如 00:20）
    *   倒计时结束后自动关闭窗口

#### 2.1.2 强制/非强制休息模式

*   **功能描述**: 用户可选择强制休息或非强制休息模式
*   **具体要求**:
    *   **强制模式**：休息期间无法直接关闭窗口，只能通过输入密码提前结束或等待倒计时结束
    *   **非强制模式**：可通过 Alt+F4 提前结束休息窗口，可通过"跳过休息"按钮提前结束休息窗口
    *   非强制模式始终显示"跳过休息"按钮，用户可随时点击此按钮结束
    *   可在设置中切换模式
    *   强制/非强制休息窗口都要求是全屏

#### 2.1.3 休息窗口

*   **功能描述**: 休息时窗口为全屏无边框显示，支持多种显示模式
*   **具体要求**:
    *   **模式1 - 全屏图片**:
        *   程序内置 5 张默认护眼图片
        *   支持用户自定义图片文件夹
        *   休息期间多张图片随机轮播，每次休息从随机图片开始，不要每次都从第一张开始
        *   轮播时间间隔：10秒
        *   显示倒计时
        *   **图片切换效果**：1-2秒淡入淡出过渡效果
        *   **自定义图片验证**: 如果用户设置的图片文件夹为空、路径无效或不包含图片，自动回退到内置图片，记录警告日志，不弹出提示
    *   **模式2 - 黑屏**:
        *   纯黑背景
        *   显示倒计时
    *   用户可在设置中选择模式
    *   默认模式：黑屏
    *   **主窗口与休息窗口之间切换时严禁出现闪烁、白屏或卡顿现象**

#### 2.1.6 休息窗口置顶

*   **功能描述**: 休息窗口始终保持在最前端
*   **具体要求**：
    *   强制模式：
        *   无法通过 Alt+F4 或其他方式关闭窗口
        *   可通过输入正确密码并点击"提前结束"按钮来提前退出休息
        *   或等待休息倒计时结束自动关闭
    *   非强制模式：可通过 Alt+F4 关闭，或点击"跳过休息"按钮关闭
    *   **强制置顶恢复**：即使被其他全屏应用（如游戏、视频播放器）覆盖，也必须定期检测并强制恢复置顶
    *   **窗口切换流畅**：主窗口与休息窗口之间切换时严禁出现闪烁、白屏或卡顿现象



#### 2.1.8 系统托盘

*   **功能描述**: 程序最小化到系统托盘
*   **具体要求**:
    *   **托盘图标状态**:
        *   正常运行时：显示默认图标
        *   暂停提醒时：显示暂停状态图标（不同的图标）
        *   图标尺寸：16x16 或 32x32 像素
    *   **托盘工具提示**:
        *   正常运行时：显示"护眼助手 - 休息倒计时中"
        *   暂停时：显示"护眼助手 - 休息倒计已暂停"
    *   **左键点击行为**:
        *   若主窗口隐藏：显示主窗口
        *   若主窗口显示：隐藏主窗口到托盘
    *   **双击行为**: 显示主窗口（无论当前状态）
    *   **右键菜单**:
        *   "显示主窗口"
        *   "暂停提醒" / "恢复提醒"（根据当前状态动态切换）
        *   分隔线
        *   "退出程序"
    *   **快捷操作**:
        *   用户可通过托盘菜单快速暂停/恢复提醒
        *   菜单项文字根据当前状态动态变化
        *   托盘菜单快速暂停/恢复提醒时，主窗口同步更新
    *   关闭主窗口时最小化到托盘而非退出

#### 2.1.9 暂停提醒与恢复提醒

*   **功能描述**: 用户可暂停休息间隔倒计时，恢复后继续计时
*   **具体要求**:
    *   点击主窗口"暂停提醒"/系统托盘右键菜单"暂停提醒"后，休息间隔倒计时停止
    *   点击主窗口按钮文字变为"恢复提醒"，系统托盘右键菜单同步变为"恢复提醒"
    *   再次点击（变为"恢复提醒"）后，按钮变为"暂停提醒"，系统托盘右键菜单同步变为"暂停提醒"，继续倒计时
    *   **主窗口与系统托盘右键菜单的"暂停提醒"/"恢复提醒"状态与文字要保持一致**


#### 2.1.5 密码保护

*   **功能描述**: 强制休息模式下需要输入密码才能提前结束休息
*   **具体要求**:
    *   首次运行程序时强制设置初始密码（对话框可关闭，不输入密码则退出程序）
    *   设置初始密码时需要输入两次密码进行确认
    *   密码长度：4-20个字符，需在配置文件中加密存储
    *   支持在设置中修改密码（需验证旧密码）
    *   输入错误密码时显示提示；
    *   如果3次输入错误，显示重置按钮，提示点击重置按钮会重置配置文件内容并退出程序
    *   密码错误计数器：每次休息时弹出休息窗口时重置为0




#### 2.1.10 退出确认

*   **功能描述**: 用户退出程序时显示确认对话框
*   **具体要求**:
    *   点击底部"退出程序"按钮时显示
    *   或右键托盘菜单选择"退出程序"时显示
    *   提供"取消"和"确定退出"两个按钮
    *   点击"确定退出"后直接退出程序，无需复选框确认

#### 2.1.10 重置确认

*   **功能描述**: 用户忘记密码时显示确认对话框
*   **具体要求**:
    *   点击主界面"忘记密码"按钮时显示
    *   或休息界面强制模式，输错3次密码时显示
    *   提供"取消"和"确定重置"两个按钮
    *   主要提示：此操作将清空所有设置，包括密码、休息间隔时长、休息时长等！
    *   点击"取消"，后直接退出
    *   点击"确定重置"后直接重置配置文件为默认值，退出程序，无需复选框确认

#### 2.1.11 日志级别设置

*   **功能描述**: 用户可设置日志记录级别，控制日志详细程度
*   **具体要求**:
    *   支持的日志级别：DEBUG、INFO、WARN、ERROR
    *   默认级别：INFO
    *   可在主窗口设置中修改
    *   修改后立即生效，无需重启程序
    *   日志级别影响日志文件和控制台输出的详细程度

#### 2.1.12 单例模式

*   **功能描述**: 确保同一时间只能运行一个程序实例
*   **具体要求**:
    *   程序启动时检测是否已有实例运行
    *   如果已有实例运行，新启动的实例立即退出，并激活已运行实例的主窗口
    *   防止配置文件冲突和资源竞争
    *   用户尝试启动第二个实例时，无提示，直接激活已运行窗口


#### 2.1.7 开机自启动

*   **功能描述**: 支持程序开机自动运行
*   **具体要求**:
    *   在设置中提供开关选项
    *   默认状态：关闭


### 2.2 界面需求

*   窗口标题、按钮、标签、提示信息等均使用中文

*   日志消息和错误提示使用中文

*   **尺寸**: 约 600x650 像素
*   **样式**: 简洁现代风格，护眼配色
*   **可调整大小**: 否
*   **可最大化**: 否
*   **位置记忆**:
    *   **程序启动**: 始终居中显示，不记忆上次的窗口位置

| 场景               | 行为                     |

| **隐藏到托盘再恢复**     | 记忆隐藏前的窗口位置，恢复时显示在原位置   |
| **休息窗口弹出前主窗口显示** | 休息结束后恢复到原来的位置和大小       |
| **休息窗口弹出前主窗口隐藏** | 休息结束后保持隐藏状态            |
| **多显示器环境**       | 如果保存的位置在当前屏幕外，使用默认居中位置 |

#### 2.2.2 休息窗口（全屏）

##### 休息窗口显示与退出行为

*   **进入休息窗口**：
    *   主窗口正常工作时：显示休息窗口，休息结束后恢复主窗口
    *   主窗口隐藏到托盘时：显示休息窗口，休息结束后保持主窗口隐藏状态
*   **退出休息窗口后的状态保持**：
    *   如果休息窗口弹出前主窗口是显示状态 → 休息结束后恢复显示主窗口
    *   如果休息窗口弹出前主窗口是隐藏状态（在托盘） → 休息结束后保持隐藏，不自动显示主窗口

##### 图片模式

*   **背景**: 全屏护眼图片（内置或用户自定义）
*   **倒计时显示**:
    *   位置：屏幕右下角
    *   颜色：灰色（低对比度）
    *   字体：小字体（不干扰视觉放松）
    *   内容：剩余休息时间，分秒格式（如 00:20）

##### 黑屏模式

*   **背景**: 纯黑色全屏
*   **倒计时显示**:
    *   位置：屏幕右下角

    *   内容：剩余休息时间，分秒格式（如 00:20）

| 非强制模式 | 按 Alt+F4     | 关闭窗口                  |

*   **触发条件**：密码错误达到3次后才显示重置按钮

### 2.3 数据存储

#### 2.3.1 配置文件

*   **存储位置**: `与 exe 文件相同目录`
*   **存储格式**: json
*   **损坏处理**: 如果配置文件损坏或格式错误，自动备份损坏文件并恢复默认配置，弹出"配置文件已损坏，已恢复默认设置"，用户点击后关闭

**注意**：窗口位置信息不保存在配置文件中。

#### 2.3.2 内置资源

*   5 张内置护眼图片（存储在程序资源中）
*   托盘图标

#### 2.3.3 日志系统

*   **日志位置**: 与 exe 文件相同目录下的 `logs/` 文件夹
*   **日志文件命名**: `eyesharp-YYYY-MM-DD-timstamp.log`
*   **日志级别**: DEBUG, INFO, WARN, ERROR（可在设置中修改，默认 INFO）
*   **日志级别说明**:
    *   **DEBUG**: 记录所有调试信息，包括详细操作日志
    *   **INFO**: 记录常规操作信息（推荐）
    *   **WARN**: 仅记录警告和错误
    *   **ERROR**: 仅记录错误信息
*   **记录内容**:
    *   程序启动/关闭
    *   配置修改
    *   休息提醒触发
    *   密码验证成功/失败
    *   异常错误信息
    *   **日志内容人工易查看**
*   **日志轮转**:
    *   每次启动新建一个日志文件

## 6. 用户使用场景

### 6.1 场景1：首次使用

1.  用户双击启动程序
2.  程序显示"设置初始密码"对话框
3.  用户设置密码（输入两次，4-20字符），或点击取消退出程序
4.  进入主窗口，使用默认配置
5.  程序自动开始倒计时，可隐藏到托盘

### 6.2 场景2：正常休息提醒

1.  倒计时结束，程序弹出全屏休息窗口
2.  显示休息倒计时（秒）
3.  用户等待倒计时结束
4.  窗口自动关闭

### 6.3 场景3：强制休息提前结束

1.  进入强制休息模式
2.  用户输入密码
3.  点击"提前结束"按钮
4.  密码验证通过，休息窗口关闭

### 6.4 场景4：非强制模式跳过

1.  进入非强制休息模式
2.  用户点击"跳过休息"按钮
3.  休息窗口立即关闭

### 6.5 场景5：隐藏到托盘

1.  用户点击标题栏关闭按钮
2.  或点击底部"隐藏"按钮
3.  主窗口隐藏到系统托盘
4.  程序继续在后台运行

### 6.6 场景6：主窗口隐藏状态下触发休息

1.  主窗口已隐藏到系统托盘
2.  定时器倒计时结束，触发休息提醒
3.  全屏休息窗口自动弹出
4.  休息倒计时结束，休息窗口自动关闭
5.  **主窗口保持隐藏状态，不自动显示**（程序继续在后台运行）

### 6.7 场景7：退出程序

1.  用户点击底部"退出程序"按钮
2.  或系统托盘右键菜单“退出程序”
3.  弹出确认对话框
4.  选择确认，程序退出
5.  选择取消，对话框关闭

### 6.8 场景8：尝试启动第二个实例（单例模式）

1.  程序已经在运行（主窗口隐藏在托盘）
2.  用户再次双击 exe 文件
3.  程序检测到已有实例运行
4.  新启动的实例立即退出
5.  已运行实例的主窗口自动显示并激活
6.  避免配置文件冲突和资源竞争
