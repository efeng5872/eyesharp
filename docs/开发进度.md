### 测试步骤

#### 1. 首次启动测试
```bash
# 删除配置文件模拟首次启动
del config.json

# 运行程序
dotnet run
```

**预期结果**：
- 应弹出"设置密码"对话框
- 输入密码（4-20字符）
- 确认密码后点击"确定"
- 程序进入主窗口
- `config.json` 文件被创建，包含密码哈希

#### 2. 配置损坏测试
```bash
# 破坏配置文件
echo "invalid json content" > config.json

# 运行程序
dotnet run
```

**预期结果**：
- 应弹出"配置恢复"对话框
- 点击"确定"后程序继续运行
- 坏掉的配置被备份为 `config.corrupted.YYYYMMDD_HHMMSS.json`
- 使用默认配置继续运行

#### 3. 日志功能测试
```bash
# 检查日志目录
dir logs

# 查看日志文件内容
type logs\eyesharp-*.log
```

**预期结果**：
- 日志文件按日期命名：`eyesharp-2025-02-13.log`
- 时间格式为北京时间
- 包含 INFO/WARN/ERROR 等级别
- 超过30天的旧日志被自动清理

## 注意事项

1. **时间格式**：所有时间显示使用北京时间（UTC+8）
2. **密码格式**：PBKDF2-HMAC-SHA256，格式 `iterations:salt:hash`
3. **配置优先级**：exe 目录 > %APPDATA%
4. **日志保留**：30天自动清理
5. **单实例**：使用 Named Mutex
