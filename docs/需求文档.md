## 1. 项目概述

### 1.1 项目名称

eyesharp - Windows 10/11 护眼提醒程序

### 1.2 项目目标

开发一款运行于 Windows 10/11 平台的护眼软件，通过定时提醒用户休息、强制休息等功能，帮助用户养成良好的用眼习惯，预防视疲劳。

### 1.3 目标用户

*   长时间使用电脑的办公人员
*   程序开发者
*   游戏玩家
*   所有需要长时间注视屏幕的用户

***

| **休息窗口弹出前主窗口显示**  | 休息结束后恢复显示主窗口          |

| 指标          | 要求                         |
| ----------- | -------------------------- |
| **窗口切换流畅性** | 主窗口与休息窗口切换响应时间 < 100ms     |
| **无闪烁要求**   | 窗口切换时严禁出现闪烁、白屏或卡顿现象        |
| **强制置顶恢复**  | 即使被其他全屏应用覆盖，也必须定期检测并强制恢复置顶 |

### 2.2 核心功能模块

#### 2.2.1 定时提醒

**功能描述**：用户可设置休息间隔时间，到达时间后触发休息提醒。

##### 功能逻辑

| 功能项         | 说明                            |
| ----------- | ----------------------------- |
| **时间范围**    | 1-120 分钟                      |
| **默认值**     | 30 分钟                         |
| **自动开始**    | 程序启动自动开始倒计时                   |
| **继续计时**    | 休息窗口结束，自动继续倒计时                |
| **倒计时刷新规则** | 修改休息间隔时间且保存后，倒计时重新从新设置的间隔时间开始 |

##### 界面设计

| 界面元素        | 位置          | 说明                                   |
| ----------- | ----------- | ------------------------------------ |
| **休息间隔输入框** | 主窗口"时间设置"分组 | 数字输入框，设置休息间隔分钟数（1-120）               |
| **休息间隔滑块**  | 主窗口"时间设置"分组 | 滑块控件，拖动选择休息间隔分钟数，与输入框双向绑定，拖动时输入框实时更新 |
| **倒计时标签**   | 主窗口"当前状态"分组 | 实时显示距离下次休息的时间（分:秒格式）                 |

##### 行为流程

    程序启动 → 自动开始倒计时 → 显示实时倒计时 → 倒计时结束 → 触发休息提醒 → 休息结束 → 再次继续倒计时

#### 2.2.2 休息前预提醒

**功能描述**：休息开始前，通过系统托盘气泡提示提前提醒用户。

##### 功能逻辑

| 功能项 | 说明 |
|--------|------|
| **预提醒时间** | 默认在休息开始前30秒和10秒各提醒一次 |
| **提醒方式** | 系统托盘气泡提示（BalloonTip） |
| **提醒内容** | "将在30秒后开始休息，请做好准备" / "将在10秒后开始休息" |
| **开关控制** | 支持开启/关闭预提醒功能 |
| **可配置性** | 预提醒时间间隔可配置（秒级） |

##### 界面设计

| 界面元素 | 位置 | 说明 |
|----------|------|------|
| **预提醒开关** | 主窗口"其他设置"分组 | 复选框，控制是否启用预提醒 |

| 指标         | 要求                     |
| ---------- | ---------------------- |

| **重复启动处理** | 新启动的实例立即退出，并激活已运行实例的主窗口     |
| **用户提示**   | 弹出对话框提示已有程序实例运行，用户点击"确定"后激活已运行窗口 |
| **目的**     | 防止配置文件冲突和资源竞争               |

##### 行为流程

    用户双击exe → 检测是否已有实例运行
                  → 是：新实例弹出对话框"程序已在运行" → 用户点击"确定" → 新实例退出，激活已运行实例主窗口
                  → 否：正常启动程序

##### 界面设计

**程序已在运行对话框**：

    ┌─────────────────────────────────────┐
    │          提示                        │
    ├─────────────────────────────────────┤

    │      护眼助手已在运行中。           │
    │      将激活已运行的窗口。           │
    │                                     │
    │              [确定]                 │
    └─────────────────────────────────────┘

|-----------|------|

| **自动关闭** | 可选：5秒后自动关闭并执行相同操作 |

#### 2.2.13 锁屏处理

**功能描述**：支持三种锁屏处理策略，用户可根据需求选择锁屏时的行为。

##### 功能逻辑

| 策略 | 标识值 | 主倒计时期间锁屏 | 休息期间锁屏 |
|------|--------|-----------------|-------------|
| **正常显示** | normal | 与锁屏无关，主倒计时到点即进入休息流程 | 休息倒计时继续，正常完成休息 |
| **暂停倒计时** | pause | 暂停主倒计时，解锁后恢复 | 休息倒计时继续，正常完成休息 |
| **跳过休息** | skip | 仅当主倒计时到点时处于锁屏：跳过本次休息，解锁后开始新主倒计时 | 休息倒计时继续，正常完成休息 |

##### 各策略详细行为

**策略1：正常显示（normal）**

| 场景 | 行为 |
|------|------|
| 主倒计时期间锁屏 | 与锁屏无关，主倒计时到点即进入休息流程 |
| 休息期间锁屏 | 休息倒计时继续运行，正常完成休息 |
| 解锁后 | 不触发额外补偿逻辑，流程按原规则继续 |

**策略2：暂停倒计时（pause）**

| 场景 | 行为 |
|------|------|
| 主倒计时期间锁屏 | 暂停主倒计时，状态变为"已暂停（锁屏）" |
| 休息期间锁屏 | 休息倒计时继续运行，正常完成休息 |
| 解锁后 | 恢复主倒计时，状态变回"运行中" |

**策略3：跳过休息（skip）**

| 场景 | 行为 |
|------|------|
| 主倒计时期间锁屏 | 仅当主倒计时到点时处于锁屏：不弹出休息窗口，进入“等待解锁”状态 |
| 休息期间锁屏 | 休息倒计时继续运行，正常完成休息 |
| 解锁后 | 若处于“等待解锁”状态，直接开始新主倒计时，不显示休息窗口 |

##### 界面设计

**锁屏处理设置**（主窗口"其他设置"分组）：

```
┌─────────────────────────────────────────┐
│  锁屏处理                               │
│                                         │
│  ◉ 正常显示（与锁屏无关，主倒计时到点即进入休息） │
│  ○ 暂停倒计时（主倒计时锁屏暂停，解锁自动恢复）   │
│  ○ 跳过休息（仅到点仍锁屏才跳过，解锁后重启倒计时） │
│                                         │
└─────────────────────────────────────────┘
```

| 界面元素 | 位置 | 说明 |
|---------|------|------|
| **锁屏处理选项组** | 主窗口"其他设置"分组 | 三个单选按钮，默认选中"暂停倒计时" |

##### 配置保存

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `lockScreenBehavior` | string | "pause" | 锁屏处理方式："normal"、"pause"、"skip" |

##### 使用场景示例

**场景1：正常显示策略**
- 用户正在工作，距离休息还有5分钟时锁屏离开
- 锁屏期间倒计时继续运行，到点即进入休息流程
- 解锁后不触发额外补偿逻辑，按原流程继续

**场景2：暂停倒计时策略**
- 用户需要临时离开电脑（如开会、接电话），锁屏前暂停倒计时
- 锁屏期间主倒计时暂停，休息倒计时不受影响
- 解锁后，主倒计时自动恢复，继续倒计时

**场景3：跳过休息策略**
- 用户锁屏时间较长（如午休），不希望锁屏期间弹出休息窗口
- 仅当主倒计时到点时处于锁屏，才跳过本次休息并进入等待解锁
- 解锁后直接开始新倒计时
